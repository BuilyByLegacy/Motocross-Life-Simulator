# GitHub Milestones

Use milestones to group issues by production outcome. Epics can span milestones, but each issue should target the milestone where its Definition of Done is expected to be met.

## 1. Legacy Bible v1.0

**Goal:** Establish the project’s foundational design vision, principles, terminology, and emotional pillars.

**Included work:**

- Core design principles and studio language.
- High-level system overview.
- Initial campaign, world, family, and motocross authenticity notes.
- Research priorities and unanswered questions.

**Exit criteria:** The design bible is coherent enough that contributors and AI coding agents can make aligned recommendations without inventing the game’s direction.

## 2. Core Engine Specs

**Goal:** Convert major simulation ideas into implementation-ready technical and design specifications.

**Included work:**

- Core Simulation Engine specs.
- Memory, Relationship, Story, Calendar, World, Family/Economy, Asset, and Competition interfaces.
- Save/load and data ownership expectations.
- MVP boundaries for each engine.

**Exit criteria:** Every MVP-critical engine has clear responsibilities, data requirements, integration points, and open risks.

## 3. MVP Vertical Slice

**Goal:** Define and build the smallest playable experience that proves the game’s promise.

**Included work:**

- One focused player journey.
- Core calendar loop.
- A limited family/economy loop.
- One race/event flow.
- Minimal memory and relationship callbacks.
- Basic phone or event planning interface if included in scope.

**Exit criteria:** A player can complete the vertical slice and experience choice, consequence, preparation, racing, and memory creation.

## 4. First Playable

**Goal:** Expand the MVP into a broader playable foundation suitable for internal evaluation.

**Included work:**

- More complete UI flow.
- Expanded event, training, garage, and marketplace interactions.
- Initial parent/rider campaign structure.
- Basic save/load reliability.

**Exit criteria:** The game can be played across multiple in-game weeks with stable progression and meaningful decisions.

## 5. Alpha

**Goal:** Reach feature-complete status for the planned core experience, even if tuning and content are unfinished.

**Included work:**

- Full core system integration.
- Representative content across major systems.
- Complete primary progression path.
- Known technical debt and bugs triaged.

**Exit criteria:** All core features are present, major workflows are testable, and remaining work is primarily content, balance, polish, and bug fixing.

## 6. Beta

**Goal:** Stabilize the experience for external testing and feedback.

**Included work:**

- Bug fixing and balance.
- UX clarity improvements.
- Performance and save/load validation.
- Content coverage for expected beta scenarios.

**Exit criteria:** No known critical blockers remain, beta feedback can be collected effectively, and the game communicates its systems clearly.

## 7. Early Access

**Goal:** Prepare the game for a public early access release with a stable, honest, expandable feature set.

**Included work:**

- Release-ready onboarding.
- Public-facing known issues and roadmap.
- Stable core loop and progression.
- Telemetry or feedback process if applicable.

**Exit criteria:** The game is safe to release publicly as an in-development product and has a clear update path.

## 8. Version 1.0

**Goal:** Deliver the complete launch promise for Motocross: Chasing the Dream.

**Included work:**

- Finished core campaigns and systems.
- Launch-quality content, tuning, and UI.
- Final bug fixing, optimization, and accessibility review.
- Documentation and support materials.

**Exit criteria:** The game meets the intended launch scope, quality bar, and emotional thesis: Build memories, not mechanics.
