# Recommended GitHub Labels

Use labels consistently so humans and AI coding agents can understand issue type, priority, area, and workflow status at a glance.

## Type

| Label | Description | Color |
| --- | --- | --- |
| `type: epic` | Large parent issue that groups related design, research, or implementation work. | `#5319E7` |
| `type: design` | Design task, spec, system definition, UX flow, or content structure. | `#1D76DB` |
| `type: research` | Research task needed before design or implementation decisions. | `#0E8A16` |
| `type: implementation` | Approved build task ready for engineering work. | `#FBCA04` |
| `type: bug` | Incorrect, broken, or confusing behavior that should be fixed. | `#D73A4A` |
| `type: tech-debt` | Refactor, cleanup, tooling, or maintainability work. | `#BFDADC` |
| `type: documentation` | Documentation-only task or documentation update. | `#0075CA` |

## Priority

| Label | Description | Color |
| --- | --- | --- |
| `priority: p0-critical` | Blocks current milestone, data integrity, release, or major playability. | `#B60205` |
| `priority: p1-high` | Important work that should be addressed soon for the active milestone. | `#D93F0B` |
| `priority: p2-medium` | Valuable work that can be scheduled after higher-priority blockers. | `#FBCA04` |
| `priority: p3-low` | Nice-to-have, future-facing, polish, or non-blocking work. | `#C2E0C6` |

## Status

| Label | Description | Color |
| --- | --- | --- |
| `status: needs-research` | Requires research before confident design or implementation work can proceed. | `#0E8A16` |
| `status: needs-design` | Requires design decisions or spec detail before development. | `#1D76DB` |
| `status: ready-for-dev` | Design and acceptance criteria are clear enough for implementation. | `#0E8A16` |
| `status: blocked` | Cannot move forward until a dependency, decision, or external issue is resolved. | `#B60205` |
| `status: in-review` | Awaiting design, code, documentation, or product review. | `#FBCA04` |
