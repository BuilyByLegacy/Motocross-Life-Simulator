# Recommended GitHub Labels

Use labels consistently so humans and AI coding agents can understand issue type, priority, area, and workflow status at a glance.

## Type

| Label | Description | Color |
| --- | --- | --- |
| `type: epic` | Large parent issue that groups related work. | `#5319E7` |
| `type: design` | Design task, spec, system definition, UX flow, or content structure. | `#1D76DB` |
| `type: research` | Research needed before design or implementation decisions. | `#0E8A16` |
| `type: implementation` | Approved build task ready for engineering work. | `#FBCA04` |
| `type: bug` | Incorrect, broken, or confusing behavior. | `#D73A4A` |
| `type: tech-debt` | Refactor, cleanup, tooling, or maintainability work. | `#C5DEF5` |

## Priority

| Label | Description | Color |
| --- | --- | --- |
| `priority: critical` | Blocks major work or breaks core behavior. | `#B60205` |
| `priority: high` | Important for current milestone or core experience. | `#D93F0B` |
| `priority: medium` | Valuable but not immediately blocking. | `#FBCA04` |
| `priority: low` | Nice-to-have, polish, or future consideration. | `#0E8A16` |

## Workflow status

| Label | Description | Color |
| --- | --- | --- |
| `status: needs-triage` | Needs owner review and scope confirmation. | `#EDEDED` |
| `status: needs-research` | Requires facts or references before decisions. | `#BFDADC` |
| `status: needs-design` | Requires approved design before implementation. | `#D4C5F9` |
| `status: ready` | Clear enough to begin. | `#0E8A16` |
| `status: blocked` | Waiting on dependency or decision. | `#B60205` |
